## 🏗️ 技术架构总结

### 系统架构

```
┌─────────────────────────────────────────────────┐
│              前端 (React + Vite)                 │
│  7个页面 + 组件库 + 服务层 (2,000+ 行)          │
└──────────────┬──────────────────────────────────┘
               │ HTTPS
               ↓
┌─────────────────────────────────────────────────┐
│         Nginx 反向代理 (负载均衡)               │
└──────────────┬──────────────────────────────────┘
               │
       ┌───────┴────────┐
       ↓                ↓
┌─────────────┐   ┌─────────────┐
│ Python API  │   │ Node.js API │
│ FastAPI     │   │ Express     │
│ 8000端口    │   │ 3000端口    │
│ (21端点)    │   │ (可选)      │
└──────┬──────┘   └─────────────┘
       │
       ↓
┌─────────────────────────────────────┐
│    SQLite 数据库 (soundscape.db)    │
│  7个表 + 关系 + 索引                 │
└─────────────────────────────────────┘
       │
       ↓
┌─────────────────────────────────────┐
│       外部 API 服务                  │
│  - OpenAI Whisper (语音转文字)      │
│  - OpenAI GPT-4 (情绪分析)          │
│  - OpenAI TTS (文字转语音)          │
└─────────────────────────────────────┘
```

### 关键设计决策

| 决策 | 原因 | 优势 |
|------|------|------|
| SQLite vs PostgreSQL | 1GB内存限制 | 无需单独DB服务器 |
| FastAPI vs Django | 轻量级+异步 | 高性能+低内存 |
| React vs Vue | 生态成熟 | 组件库丰富 |
| OpenAI API vs 本地模型 | 服务器能力有限 | 质量好、更新快 |
